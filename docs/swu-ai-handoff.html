00 · Overview

What we're building, and why it's different

Forcetable's AI is weak because it's greedy: it optimises the current move with no model of consequences. It over-commits resources, ignores what the opponent holds, and paints itself into corners. The goal here is an opponent with lookahead and threat modelling — one that gets punished in simulation for leaving itself exposed, and improves the more it's played.

  • Standalone app that reuses dmgCtrl's deck-import and card-lookup patterns.
  • Lives at dmgctrl.app as a desktop page, kept separate from the PWA until the desktop experience is solid.
  • Filed in the docs alongside the SWU rules as the reference build plan.
  • Runs and is developed locally — Windows + WSL2 + VS Code, Docker for infrastructure.
01 · Scope

Locked decisions

Sealed is the deliberate choice: a smaller ~250-card pool and 30-card minimum decks, with gameplay that emphasises unit trading over deep combo heuristics. That makes the rules engine and AI more tractable, cheaper to train, and differentiates the app from Forcetable and constructed-focused tools.

02 · Architecture

Layers of the machine

The heart of the app is a pure rules engine: (gameState, action) => newGameState. All validation lives in the legal-move generator; the resolver only applies legal actions. Everything else — UI, AI, storage — sits around that core. The AI runs in a Web Worker so the board stays responsive while it thinks.

Main thread

Board UI & input

Renders game state, surfaces the active player's legal moves, shows the game log. Sends actions in, gets new state back.

Rules engine

State · moves · resolver

Game-state schema, legal-move generator, action resolver, win check. The single source of truth for what is legal and what happens.

Web Worker

AI decision layer

Heuristic scorer → MCTS → LLM evaluator. Off the main thread so search never blocks the UI.

External

Data & models

SWUDB card detail, ProtectThePod deck JSON, local Ollama model, optional Claude API fallback.

03 · AI approach

From random to reasoning

The opponent is built up a ladder — each rung is independently shippable and beats the one below. The fix for Forcetable's core flaw arrives at MCTS (moves that lose future turns get punished in simulation) and sharpens with the LLM's explicit threat modelling.

RUNG 0
Random legal move
Proves the engine works end-to-end. Instantly beatable.
RUNG 1
Greedy heuristic scorer
Scores moves on damage, board presence, resource efficiency, base-HP swing. Already better than Forcetable.
RUNG 2
Sealed-tuned heuristics
Weights unit-trade value and tempo — where Sealed games are actually decided.
RUNG 3
MCTS in a Web Worker
Monte Carlo Tree Search with the heuristic for rollouts. Lookahead punishes over-commitment — solves the "paints itself into a corner" problem.
RUNG 4
Local LLM evaluator (Ollama)
Replaces noisy rollouts with a position score + threat model on the top-N candidates only. Q4 Mistral 7B / Llama 3.1 8B on the RTX 2080.
RUNG 5
Claude API fallback + self-play training
Optional higher-quality evaluation, plus an overnight fine-tuning loop that makes the local model SWU-specialised over time.
Performance envelope · RTX 2080 8GB

Q4 7–8B inference runs at roughly 30–50 tok/s, so a position evaluation is ~10–20s per call. Budget: heuristics drive the wide/deep tree for free; the LLM is called once per turn on the shortlisted candidates only, behind an "AI is thinking…" indicator. Fine-tuning (Unsloth + LoRA) is feasible but an overnight job — treat it as a long-horizon goal, not an inner-loop.

04 · Storage

Progressive persistence

Start with the simplest thing that works and add infrastructure only when a specific pain demands it. Each tier is introduced at the epic that needs it — nothing dockerised until the AI and training loops make it worthwhile.

TierHoldsIntroducedCost
localStorageDeck selection, preferences, active-game backupEpic 1Free
IndexedDB (Dexie)Card catalogue JSON + thumbnail blobsEpic 1Free
SQLite (Drizzle)Completed game records, decks, movesEpic 2Free
Redis (Docker)MCTS position-evaluation cacheEpic 5 · optionalFree
InfluxDB + Grafana (Docker)Win rates, think time, training metricsEpic 7Free · local
Why not localStorage for cards

~2,250 cards at 1–2KB each is 3–4MB of JSON before a single image — past the ~5MB localStorage ceiling. IndexedDB holds the JSON plus low-res WebP thumbnails as blobs (~10–15KB each), stays queryable, and works offline. Full art is fetched on demand, never cached. Running Influx and Grafana locally in Docker sidesteps every cloud free-tier limit.

▸ ARM T5.2–5.3 heuristics ────────────▸ beats Forcetable

E1–E4 plus T5.1 is the true MVP: a game you can actually play, even with a random opponent. Everything after that is opponent quality.

08 · Model strategy

Where to spend the capable model

The model isn't the bottleneck — rules-engine correctness (Epic 2) is. That's the densest, most interdependent work, and the one place a Mythos-class model's advantage genuinely compounds. Strict TDD keeps either model honest and narrows the gap everywhere else.

Recommendation

Point the Fable 5 window at Epic 2 specifically — the schema, legal-move generator and action resolver. Let Opus 4.8 handle the scaffold, data layer and UI tickets, where it's already as strong and half the cost. Don't reorganise the plan around having Fable; the tight test loop means you'd get ~90% of the same result on Opus.

If you do adopt Fable: it carries a mandatory 30-day data-retention requirement (Opus offers zero-retention). A non-issue for a hobby project, but a conscious tick.

09 · Hardware

RTX 2080 8GB — what fits

TaskFeasibilityNote
Q4 7–8B inferenceGood~30–50 tok/s; ~10–20s per evaluation call
LoRA fine-tune, 7B (Unsloth)TightFeasible but an overnight job
Full-precision 8B / any 13B+NoExceeds 8GB VRAM

Quantisation is the enabler: Q4 trades a little reasoning quality for a big memory saving — an acceptable deal for a model that will be fine-tuned on SWU specifically. A local model that has "played" thousands of Sealed games is a different proposition from a general-purpose one doing its best cold.
